# Букварь с количеством.

## 1. Задание.

> Разработать программу для работы с «букварем», представляющим буквы алфавита и
> их количество. Реализовать функционал для добавления букв, подсчёта их количества,
> проверки возможности составления слов и выполнения операций с буквами.

## 2. Архитектура программы.

Программа состоит из 2 классов и основной подпрограммы. В классах описывается логика букваря, а в основной подпрограмме
работа с данными классами.

### Классы:

1. Класс Letter - класс, который описывает определенную букву. Полями данного класса являются буква(char symbol) и 
количество таких букв в букваре(int count).

2. Класс Dictionary - класс, который описывает букварь. Полями данного класса являются список из букв, которые и
являются элементами словаря(Letter [] dictionary) и переключатель с истинным значением, который пригодится для описания
первого метода данного класса.

## 3. Методы.

### Методы класса Letter:

1. Конструктор для создания буквы. Данный конструктор принимает на вход два аргумента: букву и их количество. Полученный данные становятся полями symbol и count соответственно.

2. Геттеры. Прописаны для каждого поля данного класса.

3. Сеттеров не были реализованны, так как изменять поля в данной задаче не обязательно, однако для некоторых задач необходимо увеличивать или уменьшать поле count на единицу. Для этого были написаны методы increaseCount и decreaseCount соответственно.

#### Конструктор и дополнительные методы и функции.

Прежде чем приступить к выполнению поставленных задач, реализуем несколько методов и функций для более удобной работы внутри класса:

1. Конструктор для создания букваря. Конструктор ни принимает никаких аргументов. Он создает массив из элементов класса Letter, длинной 59(так как словарь будет содержать только русские и английские буквы). А далее заполняет данный массив буквами, сначала английские, затем русские.

#### Создание объекта с именем и количеством букв.

Для создания объекта уже создан конструктор, теперь поработаем над выводом данного класса на экран, а именно над методом toString.
В задаче продемонстрированные 2 варианта вывода класса на экран:

1. «aaaaa c kkk»
2. «a-5; c-1; k-3»

Я предполагаю, что пользователь сам должен будет выбирать какой из выводов он хочет видеть. Для этого нам и нужен flag. Он по умолчанию равен true(первый вариант вывода).
Метод пробегается по массиву dictionary и в зависимости от flag добавляет к ответу необходимый вид вывода буквы.

#### Добавление буквы.

Для добавления буквы был реализован метод addLetter. Во время работы мне показалось не оптимальным за один вызов функции добавлять всего одну букву, поэтому метод принимает на вход строку и добавляет в массив все буквы из этой строки, подходящие для словаря.

#### Подсчёт количества определённой буквы.

Для подсчета количества определенной буквы был реализован метод countOfLetter. Он принимает на вход букву, далее с помощью find_index получает индекс данной буквы и потом обращается к массиву и через геттер возвращает количество данной буквы.

#### Подсчёт общего количества букв.

Для подсчёта общего количества букв был реализован метод totalAmount. Он проходится по массиву, у каждого элемента запрашивает геттер количества букв и складывать их между собой, а затем полученную сумму метод возвращает.

#### Сборка слова из букв.

Для сборки слова из букв был реализован метод negativePossibleWord. Данный метод проходится по каждой букве слова, если их количество не равно нулю то добавляет данную букву и уменьшает количество данной буквы на единицу.

#### Проверка возможности составления слова.

Для проверки возможности составления слова был реализован метод isWordPossible. Данный метод принимает на вход слово, далее создает новый словарь и добавляет в него буквы данного слова. Далее алгоритм пробегается по буквам исходного словаря и поочередно сравнивает количество букв исходного массива с новым. Если в какой-то буквы в новом словаре окажется больше, то сразу возвращаем false. Если false ни разу не был вызван возвращаем true.

#### Сборка слова без изменения массива.

Для сборки слова без изменения массива был реализован метод positivePossibleWord. Данный метод работает точно также как метод negativePossibleWord, но он не уменьшает количество букв.

#### Прочитать слово, если возможно.

Для прочтения слова, если возможно был реализован метод fullWord. Данный метод сначала проверяет возможно ли составить слово, а затем пробегается по каждому элементу слова и проверяет являться ли оно элементом массива. Если все проверки пройдены возвращает изначальное слово, в противном случае пустую строку.

#### Подсчёт количества возможных слов.

Для подсчёта количества возможных слов был реализован метод numberPossibleWords. Метод создает новый словарь буквами, которого являются буквы данного на вход метода слова. Далее мы проходимся по каждой букве, у которой count не равен нулю. Затем мы находим минимальное среди частных между количеством букв исходного массива и нового. Полученное частное и будет ответом.
